# Release Guide

## Current status

No application exists and no release is planned.

## Future release gates

A public release will eventually require:

- Approved product scope
- Passing required automated checks
- Passing critical end-to-end journeys
- Independent audit
- Preview-environment verification
- Founder approval
- Secure production configuration
- Database migration verification, when applicable
- Monitoring and error reporting
- Backup and rollback plan
- Privacy, terms, support, and customer-facing readiness
- Production smoke test

A deployment URL alone does not mean the product is publicly ready.
