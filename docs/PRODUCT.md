# Product

## Status

An **initial proposed product direction** now exists, but the product definition is not yet locked.

Kidora is currently envisioned as a preschool operating system covering the journey from parent enquiry and admission through student records, fees, attendance, communication, teacher updates, and child progress.

The initial roadmap is recorded in `docs/ROADMAP.md`.

## Proposed users

- Parents and guardians
- School administrators
- Teachers
- School owner or management

## Proposed first product slice

```text
Parent scans QR
→ Completes child registration
→ Receives confirmation
→ Admin reviews application
→ Admin updates status
→ Admin contacts parent
```

This first slice is intended to validate Kidora's real admission workflow before broader school-management modules are built.

## Important boundary

The roadmap is a direction, not approval to build the complete product.

The following still require founder approval and focused discovery:

- Exact parent problem and workflow
- Required registration fields
- Consent and privacy wording
- Admin statuses and follow-up process
- Final scope of the first release
- Architecture and technology choices
- Whether the product is only for Kidora or later supports multiple schools

No agent may silently decide these items.

## Next product step

Complete focused discovery for Phase 1 and create an approved OpenSpec change before implementation begins.
