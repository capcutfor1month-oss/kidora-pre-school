# Testing Strategy

## Current status

No application exists yet, so application tests are not configured.

## Future layers

When development begins, each applicable change will define:

1. **Static checks** — formatting, linting, and type checks
2. **Unit tests** — isolated logic
3. **Integration tests** — frontend, backend, database, and services working together
4. **End-to-end tests** — complete browser journeys
5. **Security tests** — authorization, isolation, validation, and secrets
6. **Independent audit** — specification, code, tests, and runtime evidence
7. **Founder usability test** — clarity, usefulness, and real-life experience

## Evidence rule

A feature is not complete because an agent says it is complete. Required checks and user journeys must provide evidence.

## Pipeline validation

Before application development, CI only validates the repository's pipeline documents and anti-duplication rules.
